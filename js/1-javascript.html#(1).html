The emergence of JavaScript

In 1990s, Marc Andreessen, the founder of Netscape, believed that:

HTML needed a “glue language” that was easy to use to assemble components such as images and plugins.

  • In 1995, Brendan Eich developed a script language called LiveScript.
  • In 1995, LiveScript was renamed to JavaScript.

Many variants, some vendor specific



Javascript & ECMAScript

You'll come across references to ECMAScript

ECMAScript (or ES) is a trademarked scripting-language specification standardized by Ecma International in ECMA-262 and ISO/IEC 16262. It was created to standardize JavaScript, so as to foster multiple independent implementations.

ECMAScript has remained the best-known implementation of JavaScript since the standard was first published, with other well-known implementations including JScript and ActionScript.
….
While both JavaScript and JScript aim to be compatible with ECMAScript, they also provide additional features not described in the ECMA specifications.

Example of JavaScript Syntax

originalFont = null   // Global - place to save original
fontChanges = Array();

function setFont(whichFont){
   if (originalFont == null) {
      originalFont = document.body.style.fontFamily;
   }
   document.body.style.fontFamily = whichFont;
   fontChanges.push(whichFont);
}

function showChanges(){

      alert("No changes")

      s = ''
      for (i=0; i<fontChanges.length; i++) {
        s = s + fontChanges[i] + '<br>'
      document.getElementById('changeList').innerHTML = s
      }
   }
}

Javascript and Semicolons

Semicolons terminate statements BUT JavaScript doesn't require them.

Semicolons are assumed at the end of a line

Usually this is correct, but can introduce ambiguity 

a = b + c
(d + e).upper()

// Is treated as:
// a = b + c(d + e).foo();

NOT THE SAME AS

a = b + c;
(d + e).upper()

It's an obscure point, but generally using semicolons is safer

For details, SKIM http://inimino.org/~inimino/blog/javascript_semicolons
– "SKIM" - i.e. NOT required reading

Logical (Boolean) Operators

Uses the 'C' Style for and, or and not

  • && is AND
  • || is OR
  • ! is NOT

e.g. 

if   x > 9 and not (b == 7 or x == 0):  # in Python

becomes 

if (  x > 9 && !(b == 7 || x == 0)   )


Ternary Operator

This is like an if expression

If contains a condition and two values: 

condition ? value if True : value if False

e.g. 

x = (reply == "Hello") ? 99 : 0

is the same as: 

if (reply == "Hello") 
   x = 99
else
   x = 0

Object Coercion

If an object is compared with a number or string, JavaScript attempts to return the default value for the object.

An object is converted to a primitive String or Number value, using the .valueOf() or .toString() methods of the object.

If this fails, a runtime error is generated.
